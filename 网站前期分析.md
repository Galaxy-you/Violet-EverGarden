# 紫罗兰永恒花园主题网站前期分析

## 1. 课程/项目设计要求拆解（来自 `主题网站设计要求.md`）

### 核心目标
- 完成从页面结构到视觉风格再到轻交互的完整实现能力展示。
- 网站必须同时具备：视觉吸引力、真实内容、可读性、交互友好性。
- 体量要求：`4-6` 个页面，预估 `20-30h` 工作量。

### 硬性要求
- 站点形态：`4-6` 页主题式静态独立网站，页面逻辑连贯，形成内容闭环。
- 技术栈：`HTML5 + CSS3`（必做）+ 适量 `JavaScript`。
- 可使用轻量 CSS 框架，但禁止直接改成熟建站模板。
- 必须适配多端（至少桌面/平板/手机）。
- 全站视觉系统要统一（色彩、字体、版式、图标不能割裂）。

### 页面与交互要求
- 页面结构清晰、导航可用、无死链、无跳转混乱。
- 需要原创/个性化视觉元素（如 logo、banner、主题图标等）。
- 细节交互建议覆盖：hover、按钮状态、加载/过渡、图片加载体验。

### 加分方向
- 响应式细化到三端。
- 工程优化：CSS 变量、可复用 JS、图片压缩、加载速度优化。

2. `世界观 World`
3. `角色 Character`
4. `故事 Story`
5. `音乐 Music`
6. `制作与外传 Staff & Side Story`

### 每页功能目标
- 首页：主视觉、主题标语、分流入口。
- 世界观：地名/场景图组 + 简述。
- 角色：角色卡片、关系线索、CV。
- 故事：时间线/章节式叙事。
- 音乐：专辑封面、曲目亮点、情绪标签。
- 制作与外传：制作班底 + 外传内容入口，形成内容闭环。

## 7. 交互与工程基线（与设计要求对齐）

### 必做交互
- 导航高亮与滚动定位
- 卡片 hover 层次变化
- 图片懒加载和淡入
- 页面或区块过渡动画（轻量）

### 工程建议
- 建立统一设计令牌：`--color-*`、`--font-*`、`--space-*`
- JS 按功能拆分（导航、动画、数据渲染）
- 图片按页面分目录并做压缩版本
